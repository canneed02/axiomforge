## 1. Falsifiable Claim

A retrieval-augmented generation (RAG) system with explicit provenance tracking and confidence calibration reduces hallucinated citations in autonomous scientific literature review by ≥40% compared to standard vector-retrieval baselines, measured by human-verified citation accuracy on a held-out set of 500 papers from arXiv CS 2023-2024.

---

## 2. Experiment / Proof Harness

**System:** `research-memory-v0` — a modular harness with:
- **Retrieval:** Hybrid dense-sparse index (ColBERTv2 + BM25) over arXiv abstracts
- **Provenance:** Every generated claim tagged with `(paper_id, chunk_id, retrieval_score)`
- **Calibration:** Temperature-scaled confidence scores trained on historical retrieval precision

**Evaluation Protocol:**
- Generate 1,000 literature-review paragraphs (synthetic queries from paper titles)
- Human annotators verify: (a) citation exists, (b) supports claim, (c) no cherry-picking
- Primary metric: `verified_citation_rate` — fraction of claims with correct, non-misrepresented citations

**Baseline:** Standard top-k cosine similarity with GPT-4 generation.

---

## 3. Failure Modes

| ID | Failure Mode | Detection |
|---|---|---|
| **F1** | *Retrieval collapse*: High-confidence retrieval from review papers masquerading as primary sources | Check `paper_type` metadata; flag secondary-source chains |
| **F2** | *Confidence miscalibration under distribution shift*: System overconfident on out-of-domain queries (e.g., biology papers in CS index) | Track domain classifier entropy; reject if above threshold |
| **F3** | *Provenance laundering*: Model paraphrases retrieved text sufficiently to invert meaning, while preserving citation | Semantic entailment check between source chunk and generated claim; flag non-entailment |

---

Enables transparent comparison of future systems on citation fidelity rather than fluency.
