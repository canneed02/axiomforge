## 1. Falsifiable Claim

A retrieval-augmented generation (RAG) system with **temporal versioning and explicit uncertainty quantification** reduces cumulative error propagation in multi-step scientific reasoning by ≥30% compared to standard vector-store RAG, measured by ground-truth trajectory recovery on held-out scientific workflows.

---

## 2. Experiment/Proof Harness

**"MemTrace" Benchmark**

- **Dataset**: 500 synthetic scientific workflows (e.g., hypothesis → experiment → analysis → refinement) with injected ground-truth dependency graphs; 50 derived from real OpenAlex paper chains with human-verified citation graphs.
- **System**: Versioned memory store with:
  - Per-retrieval confidence scores (cosine similarity + ensemble disagreement)
  - Explicit "unknown" flagging when confidence < threshold
  - Temporal provenance chains for all retrieved claims
- **Control**: Standard RAG (single vector store, no versioning, no uncertainty flagging)
- **Metric**: Trajectory recovery F1 (ground-truth step sequence vs. executed) + hallucination rate (LLM judge + human audit on 100-sample subset)

**Code target**: Reproducible in <8 hours on single A100; all prompts and evaluation harness open-sourced.

---

## 3. Failure Modes

| Mode | Detection |
|------|-----------|
| **Confidence collapse**: Uncertainty estimates become miscalibrated after recursive self-updates (overconfident on stale entries) | Track calibration drift via held-out query set; flag if ECE > 0.15 |
| **Version explosion**: Temporal chains grow superlinearly, degrading retrieval latency and relevance | Monitor query latency + top-k recall; fail if p99 latency >2s or recall@5 <0.7 |
| **Silent contradiction**: System retrieves and acts on mutually incompatible claims without detecting conflict | Explicit contradiction detection module; log unresolved conflicts as critical errors |

---

- The benchmark dataset (synthetic + real workflows)
