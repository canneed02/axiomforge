## 1. Falsifiable Claim

A retrieval-augmented generation (RAG) system with episodic memory (storing failed hypotheses + negative results) outperforms identical systems with only success-indexed memory on novel scientific hypothesis generation, measured by 10% higher precision@k on a held-out benchmark of derivable but non-obvious conjectures in a formal domain (e.g., synthetic theorems in Metamath or TPTP).

## 2. Experiment / Proof Harness

| Component | Variant A (control) | Variant B (treatment) |
|-----------|---------------------|----------------------|
| Memory store | Successful proofs only | Successes + failed proof attempts with error traces + associated lemma context |
| Retrieval | Dense on goal state | Dense on goal state + sparse on failure pattern matching |

**Task:** 500 held-out theorems from `mathlib` dependencies not in training data. Measure: (a) proof success rate, (b) average proof length, (c) novelty of intermediate lemmas (edit distance to existing library).

**Validation:** Run 10 seeds; use paired t-test. Pre-register on arXiv before executing.

## 3. Failure Modes

| ID | Failure Mode | Detection |

| F1 | **Negative memory interference**: Failed-attempt retrieval actively misleads on structurally similar but valid new paths. Detect via ablation: measure retrieval-to-success correlation; if ρ < 0, memory is harmful. |
| F2 | **Memory bloat without decay**: Episodic store grows superlinearly, retrieval latency degrades, and relevance drops. Detect via recall@10 vs. store size curve; should plateau or degrade. |
| F3 | **Overfitting to benchmark**: System learns to pattern-match failure signatures in `mathlib` style without generalizing. Detect via cross-domain transfer to Isabelle/HOL or synthetic domain with no shared lemmas. |
