A retrieval-augmented generation (RAG) system with **episodic memory consolidation**—where intermediate research steps are compressed via gradient-based summarization and stored in a structured knowledge graph—outperforms append-only log baselines on multi-step scientific reasoning tasks by ≥15% on answer accuracy, measured after 100 sequential research steps with context window overflow.

## 2. Experiment / Proof Harness

**Task:** Open-ended hypothesis generation on the BIG-bench "Scientific Reasoning" subset, extended to 100-step trajectories.

**Implementation:**
- **Baseline:** Append-only research log with standard RAG (dense retrieval over raw step history).
- **Treatment:** Consolidation module using LoRA-tuned encoder to compress episode triples `(observation, action, outcome)` into graph nodes; retrieve via hybrid (vector + graph traversal) search.
- **Evaluation metric:** Accuracy on held-out verification questions + token efficiency (total tokens read to reach correct answer).

**Code harness:** `git` repository with deterministic seeding, Docker environment, and CI-triggered evaluation on fixed benchmark subset.

## 3. Failure Modes

| Mode | Description | Detection |
|------|-------------|-----------|
| **Catastrophic consolidation** | Compressor drops critical negative results; system repeats failed paths. | Track "regret" metric: frequency of repeated identical failed actions. |
| **Graph hallucination** | Structured storage invents non-existent relationships during retrieval augmentation. | Ground-truth edge verification against source logs; precision@k on edges. |
| **Memory bloat inversion** | Over-optimization for compression causes near-empty retrieval; degrades to context-window-only performance. | Monitor retrieval hit rate vs. effective context usage ratio. |
