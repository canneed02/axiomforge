## 1. Falsifiable Claim

A research memory system using structured provenance logging (action, observation, timestamp, confidence) enables more effective error recovery and hypothesis revision than raw text logs, measurable by task completion rate on held-out scientific benchmark tasks after injected failures.

---

## 2. Experiment/Proof Harness

**Harness:** `repro-memory-v1`

- **Baseline:** GPT-4 with unstructured text-only research logs (append-only)
- **Treatment:** Same model with structured provenance tuples `(action, observation, timestamp, confidence, dependencies)`
- **Task:** 50 synthetic scientific tasks (e.g., literature synthesis, experimental design, data analysis) with mid-execution injected failures (random tool unavailability, corrupted intermediate results)
- **Metric:** Task completion rate within token budget; secondary: time to recover from failure (measured in recovery actions)
- **Control:** Human-verified ground truth for each task; failure injection randomized and blinded

---

## 3. Failure Modes

| ID | Failure Mode | Detection |
|---|---|---|
| F1 | **Confidence drift:** Structured logs show calibrated confidence, but model ignores low-confidence entries during recovery, repeating errors | Log analysis: correlation between confidence and selection frequency |
| F2 | **Dependency hallucination:** Model invents non-existent provenance links to justify current actions; structured format masks this with false precision | Ground-truth dependency graph comparison via automated diff |
| F3 | **Compaction collapse:** Summarization of long memory chains loses failure-relevant context despite structured format | Inject known failure patterns; measure recall post-compaction |

---
