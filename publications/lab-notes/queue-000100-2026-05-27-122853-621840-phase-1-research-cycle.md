## 1. Falsifiable Claim

A retrieval-augmented generation (RAG) system with episodic memory (tracking experiment outcomes with temporal decay) outperforms a stateless RAG system on scientific hypothesis generation, measured by 30% higher precision in predicting validated experimental outcomes on held-out benchmark tasks from the Open Research Knowledge Graph (ORKG).

## 2. Experiment / Proof Harness

**Memory-OR-Bench**: A reproducible evaluation framework.

- **Dataset**: Curated subset of 10,000 scientific papers with structured experimental outcomes (hypothesis → method → result → validation status).
- **Baseline**: Stateless RAG (GPT-4 + vector DB, no memory).
- **Treatment**: Identical architecture + episodic memory module:
  - Stores (experiment_id, outcome, confidence, timestamp) tuples.
  - Retrieval weighted by recency and outcome surprise (negative log-likelihood of prediction).
- **Metric**: Precision@k of generated hypotheses against held-out validated results; secondary: recall of relevant prior failures.
- **Control**: Freeze LLM, vary only memory mechanism.

**Code**: `github.com/[org]/memory-or-bench` — Dockerized, with deterministic seeding.

---

## 3. Failure Modes

| Mode | Mechanism | Detection |
|------|-----------|-----------|
| **Memory illusion** | System retrieves confabulated "past experiments" that never occurred, reinforcing false confidence. | Audit log mismatch: compare retrieved memory IDs against ground-truth experiment registry. |
| **Negative gradient neglect** | Episodic weighting over-prioritizes recent successes; failures decay too fast, causing repeated error types. | Track failure recurrence rate; should exceed baseline if memory is functional. |
| **Retrieval interference** | Semantically similar but causally distinct experiments conflate, producing spurious analogies. | Ablation: measure precision drop when memory is disabled vs. when similar-but-irrelevant memories are injected. |

---

## 4. Minimal Publishable Artifact

**The Memory-OR-Bench Audit Log**: A structured, versioned dataset of all system runs including—per inference—(a) the query, (b) retrieved memory IDs with full content hashes, (c) generation output, (d) ground-truth label, (e) timestamp. Enables independent reproduction of any claim and post-hoc analysis of failure modes 1–3. Format: Parquet + IPFS content-addressing for immutability.
