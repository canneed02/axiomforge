**Research Memo: Autonomous Research Memory**

---

## 1. Falsifiable Claim

A retrieval-augmented generation (RAG) system with **episodic memory consolidation**—where intermediate research steps are compressed into summary embeddings at exponentially spaced intervals (1, 2, 4, 8... steps)—achieves lower cumulative regret on open-ended scientific tasks than systems with uniform sampling or no compression, measured by expert evaluation of research plans after 50 steps.

---

## 2. Experiment / Proof Harness

**Task:** Open-ended protein design benchmark (e.g., generate functional variants of lysozyme with specified properties).

**Setup:**
- Base LLM: fixed open-weight model (e.g., Llama-3-70B)
- Three conditions: (a) no memory, (b) uniform retrieval (every step), (c) exponential consolidation
- Memory: ChromaDB with `all-MiniLM-L6-v2` embeddings; consolidation uses LLM summarization
- **Metric:** Cumulative regret = Σᵢ[1 - (expert rating of step i / 10)], evaluated by 3 blinded protein engineers

**Code harness:** `git clone`able repo with `pytest` tests verifying determinism, evaluation protocol, and statistical comparison (paired t-test, α=0.05).

---

## 3. Three Failure Modes

| # | Failure Mode | Detection |
|---|-------------|-----------|
| 1 | **Consolidation collapse:** Summaries become homogenized; entropy of retrieved contexts drops below baseline within 20 steps | Measure embedding cosine-similarity variance; alarm if σ² < 0.05 |
| 2 | **Retrieval fixation:** System repeatedly fetches same 3-4 memories; Gini coefficient of retrieval frequency > 0.8 | Track access histogram per step |
| 3 | **False continuity:** Consolidated memories invent "discoveries" not in raw traces; hallucination rate > 5% via NLI against source | Automated entailment check (DeBERTa-MNLI) |

---

## 4. Minimal Publishable Artifact

**Consolidated memory trace + reproducibility bundle:** A single `.tar.gz` containing:
- `memory_log.jsonl`: timestamped (raw_memory, consolidated_memory, retrieval_query, retrieved_keys) for all 50 steps
- `environment.lock`: exact Docker image digest, dependency hashes
- `regret_scores.json`: per-step expert ratings with rater IDs

Enables: independent re-scoring, forensic analysis of failure modes, and direct comparison if claim is challenged.
