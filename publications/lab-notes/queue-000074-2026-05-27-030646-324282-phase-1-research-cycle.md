## 1. Falsifiable Claim

A retrieval-augmented generation (RAG) system with episodic memory—indexed by (hypothesis, outcome, confidence) triples—outperforms a stateless LLM on scientific hypothesis generation by ≥15% on a held-out benchmark of reproducible experimental design, measured by expert-rated novelty and feasibility scores.

## 2. Experiment / Proof Harness

**Benchmark:** `SciMem-Bench` — 500 synthetic research threads (ground-truth: known physics/chemistry results) + 50 real open problems from arXiv preprints abstracts.

**Protocol:**
- **Baseline:** GPT-4 with 5-shot prompting, no memory.
- **Treatment:** Same model + vector memory (ChromaDB) of past hypotheses, experimental outcomes, and updated confidence scores. Memory retrieval triggered by cosine similarity to current problem embedding.
- **Metric:** Expert blind rating (1–5) of generated next-experiment proposals; primary = average score, secondary = citation-verified factual accuracy.

**Validation:** Run 10 seeds; check if mean(treatment) − mean(baseline) ≥ 0.75 points (15% of scale) with p < 0.05 (Welch's t-test).

---

## 3. Failure Modes

| # | Failure Mode | Detection |
|---|-------------|-----------|
| 1 | **Retrieval collapse:** Memory over-reuses high-confidence but irrelevant past hypotheses, causing local minima in exploration. | Track retrieval diversity (unique hypothesis clusters accessed); alarm if Gini > 0.8. |
| 2 | **Confidence drift:** Self-updated confidence scores diverge from empirical success rates, creating false certainty. | Compare running confidence vs. ground-truth accuracy on synthetic threads; KL divergence > 0.5 bits. |
| 3 | **Benchmark leakage:** LLM has seen training data related to test problems, inflating "memory" gains. | Filter training corpora by arXiv ID; verify with Min-K% probing; reject if overlap > 1%. |

---
