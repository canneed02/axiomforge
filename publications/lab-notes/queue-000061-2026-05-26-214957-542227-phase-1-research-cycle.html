## 2. Experiment / Proof Harness

**Harness:** `research-memory-v0`

| Component | Specification |
|-----------|-------------|
| **Memory substrate** | Versioned append-only log of (hypothesis, protocol, outcome, confidence) tuples with cryptographic provenance |
| **Retrieval** | Embedding-based similarity + structured metadata filtering over experimental history |
| **Agent loop** | LLM planner → protocol generator → execution sandbox → outcome validator → memory commit |
| **Baseline** | Identical agent with memory retrieval disabled (randomized ablation) |
| **Benchmark** | 3 established synthetic scientific tasks: (a) symbolic regression discovery, (b) hyperparameter optimization on fixed model families, (c) materials property prediction with simulated lab costs |
| **Metric** | Coverage (unique valid configurations explored), redundancy rate (repeated equivalent experiments), convergence time to target performance threshold |

**Validation protocol:** Open-source implementation, deterministic seeding, reproducible containerized execution.

---

## 3. Three Failure Modes

| ID | Failure Mode | Detection Criterion |
|:---|:---|:---|
| **F1** | **False negative retrieval** | Semantically similar past experiment not recalled; detected by post-hoc embedding similarity between executed and historical protocols exceeding threshold without retrieval trigger |
| **F2** | **Memory poisoning** | Corrupted or overconfident historical entry propagates into future protocol generation; detected by outcome confidence calibration drift (>10% Brier score increase on held-out validation) |
| **F3** | **Exploration collapse** | Over-reliance on memory suppresses novel protocol generation; detected by entropy of generated protocol distributions declining below 20% of baseline over any 100-step window |
